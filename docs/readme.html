How To Use The Package?

The Recommended Way

  1. Add the package to your project. 
    cargo add mirth_engine_tickers
  2. Add .add_plugin(Tickers{}) to your Bevy app.
  3. You can now make use of the datatypes that this package offers.

The "I Choose Features" Way

  1. Add the package to your project with the default features disabled, then explicitly choose which features you want.
    • ticker_reflect : Will cause all ticker types to reflect. Will add the bevy_reflect dependency and allow for the Tickers plugin to be usable.
    • ticker_systems : Will turn on all systems for every ticker type. Will add the bevy_time dependency and allow for the Tickers plugin to be usable.
    • ticker_serialize : Will make it so that ticker types can be serialized and deserialized. Will enable ticker_reflect, add the serde dependency, and allow for the Tickers plugin to be usable. 
      cargo add mirth_engine_tickers --no-default-features --features ticker_reflect, ticker_systems, ticker_serialize
  2. If you added any of the features, add .add_plugin(Tickers{}) to your Bevy app.
  3. You can now make use of the datatypes that this package offers.

The "I Will Do Everything Myself" Way

  1. Add the package to your project with the default features disabled. 
    cargo add mirth_engine_tickers --no-default-features
  2. You can now make use of the datatypes that this package offers, but you will need to create ticking systems, reflection registrations, and serialization capability yourself.

What Are The Datatypes This Package Offers?

  • Tickers
    • A Bevy plugin used to activate ticker systems and reflections for ticker types.
    • Use .add_plugin(Tickers{}) inside a Bevy app to enable ticker systems and reflections.
  • Ticker
    • A class used to track the time between events.
    • Holds the following fields:
      • start_value
      • current_value
      • end_value
      • time_interval
      • is_paused
      • is_ticking_up
      • is_handling_time_spikes
      • behavior
    • Is supported by the following types:
      • TickerBehaviors
        • An enum that establishes the different types of behavior a Ticker can use.
          • Supports Looper, MutLooper, Oneshot, MutOneshot, and Freezing behaviors.
      • TickerValue
        • A trait for implementing a generic to define integer primitives a Tickercan potentially store for its boundary values and current value.
          • Supports i8, i16, and i32.
      • TickerPrecision
        • A trait for implementing a generic to define float types a Ticker can potentially use for its precision in tracking time.
          • Supports f16, f32, and f64.
      • TickerFloatBridge
        • A trait which grants the ability for f16, f32, and f64 to be passed in for the float fields of a Ticker constructor, no matter the precision a ticker is set to.
          • Eases usage of Ticker constructor methods.

What Can I Do With The Ticker Class?

DESCRIBE FEATURES THAT AREN'T METHODS OF TICKER HERE

All the following methods are available to you:

Constructors

  • new

    Used for completely defining a custom ticker.

  • new_looper

    Creates an unpaused Looper ticker that ticks current_value from the supplied starting value to the passed end value. The ticker is immutable and will loop when current_value hits either boundary, resetting back to the start. If the initial start and end values are equal, the tick direction defaults to up.

  • new_looper_custom

    Creates an unpaused Looper ticker with fully custom starting fields. The ticker is immutable and will loop when current_value hits either boundary, resetting back to start_value.

  • new_mut_looper

    Creates an unpaused MutLooper ticker that ticks current_value from the supplied starting value to the passed end value. The ticker is mutable and will loop when current_value hits either boundary, resetting back to the start. If the initial start and end values are equal, the tick direction defaults to up.

  • new_mut_looper_custom

    Creates an unpaused MutLooper ticker with fully custom starting fields. The ticker is mutable and will loop when current_value hits either boundary, resetting back to start_value.

  • new_oneshot

    Creates an unpaused Oneshot ticker that ticks current_value from the supplied starting value to the passed end value. The ticker is immutable and will assign current_value to whichever boundary it hits. stored_time resets to 0.0 once current_value reaches end_value. If the initial start and end values are equal, the tick direction defaults to up.

  • new_oneshot_custom

    Creates an unpaused Oneshot ticker with fully custom starting fields. The ticker is immutable and will assign current_value to whichever boundary it hits; stored_time resets to 0.0 once current_value reaches end_value.

  • new_mut_oneshot

    Creates an unpaused MutOneshot ticker that ticks current_value from the supplied starting value to the passed end value. The ticker is mutable and will assign current_value to whichever boundary it hits. stored_time resets to 0.0 once current_value reaches end_value. If the initial start and end values are equal, the tick direction defaults to up.

  • new_mut_oneshot_custom

    Creates an unpaused MutOneshot ticker with fully custom starting fields. The ticker is mutable and will assign current_value to whichever boundary it hits; stored_time resets to 0.0 once current_value reaches end_value.

  • new_freezing

    Creates an unpaused Freezing ticker that ticks current_value from the supplied starting value to the passed end value. The ticker begins mutable but becomes immutable once current_value hits end_value; stored_time resets to 0.0 at that point. If the initial start and end values are equal, the tick direction defaults to up.

  • new_freezing_custom

    Creates an unpaused Freezing ticker with fully custom starting fields. The ticker begins mutable but becomes immutable once current_value hits end_value; stored_time resets to 0.0 at that point.

  • new_copy

    Creates a copy of the passed ticker.

  • new_copy_with_behavior_change

    Creates a copy of the passed ticker with the only field change being the behavior, which is set to the passed TickerBehaviors value. Useful for safely turning an immutable ticker into a mutable one — copy its values into a new mutable instance, then swap the immutable original out for the copy.

Getters

  • start_value

    Returns the start_value of a ticker.

  • current_value

    Returns the current_value of a ticker.

  • end_value

    Returns the end_value of a ticker.

  • time_interval

    Returns the time_interval of a ticker — the amount of time it takes for current_value to increase or decrease by 1. A ticker has no built-in concept of "seconds" or any other unit; time_interval and stored_time are just two numbers compared inside .tick(), and the unit they represent depends entirely on what's passed into .tick().

  • stored_time

    Returns the stored_time of a ticker — the leftover remainder from the last .tick() call, not the total elapsed time since creation or reset. Mainly useful for debugging, logging, or custom structures that need to inspect or manually carry over a ticker's in-progress timing state.

  • is_paused

    Returns true if the ticker is paused, false otherwise.

  • is_ticking_up

    Returns true if a ticker is set to tick its current_value up, false otherwise.

  • is_ticking_down

    Returns true if a ticker is set to tick its current_value down, false otherwise.

  • is_handling_time_spikes

    Returns true if the ticker can change current_value by any amount greater than or equal to 0 in a single .tick() call, catching up all built-up time at once. Returns false if the ticker can only change current_value by 1 or 0 per .tick() call, no matter how much time has built up between calls.

  • behavior

    Returns the behavior type of the ticker.

Setters

  • set_start_value

    Changes start_value to the passed value. If the new start_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge. start_value itself cannot go outside V::MIN to V::MAX and will be clamped down if it does.

  • set_current_value

    Changes current_value to the passed value. current_value cannot go outside the range that start_value and end_value create and will be clamped down if it does.

  • set_end_value

    Changes end_value to the passed value. If the new end_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge. end_value itself cannot go outside V::MIN to V::MAX and will be clamped down if it does.

  • pause

    Prevents .tick() calls on a ticker from doing their job.

  • unpause

    Allows .tick() calls on a ticker to do their job.

  • tick_up

    Causes the ticker's current_value to count up.

  • tick_down

    Causes the ticker's current_value to count down.

  • start_handling_time_spikes

    Makes it so that .tick() calls on a ticker add or subtract all built-up integer time since the last .tick() call to current_value (direction depends on is_ticking_up), with any floating remainder carried over into stored_time for the next call.

  • stop_handling_time_spikes

    Makes it so that .tick() calls on a ticker only add or subtract 1 to current_value (direction depends on is_ticking_up), no matter how much time has built up between calls.

  • set_behavior

    Switches the behavior of a ticker to the passed TickerBehaviors type. Does not work for tickers with immutable behavior — use new_copy_with_behavior_change to simulate switching an immutable ticker to mutable behavior, then substitute the copy in for the original.

Equality Methods

  • is_current_at_start

    Returns true if current_value and start_value are equal to one another, false otherwise. Useful in oneshot tickers that count down to start_value, to determine if the oneshot is finished.

  • is_current_at_end

    Returns true if current_value and end_value are equal to one another, false otherwise. Useful in oneshot tickers that count up to end_value, to determine if the oneshot is finished.

  • is_start_at_end

    Returns true if start_value and end_value are equal to one another, false otherwise. Since both values can be changed after a ticker is created, this is mainly useful when the bounds of a ticker are slowly tightening and you need to detect when they've finally met.

Difference Methods

  • difference_from_start

    Returns the difference between current_value and start_value. Will only return positive numbers, including 0.

  • difference_from_end

    Returns the difference between current_value and end_value. Will only return positive numbers, including 0.

  • difference_from_start_to_end

    Returns the difference between start_value and end_value. Will only return positive numbers, including 0.

Digit Methods

  • digit_1

    Returns the digit in the ones-place of current_value. Always positive. The ones-place always exists, so there is no digit-drop-accounting version of this method.

  • digit_2

    Returns the digit in the tens-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_3

    Returns the digit in the hundreds-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_4

    Returns the digit in the thousands-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_5

    Returns the digit in the ten-thousands-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_6

    Returns the digit in the hundred-thousands-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_7

    Returns the digit in the millions-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_8

    Returns the digit in the ten-millions-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_9

    Returns the digit in the hundred-millions-place of current_value. Always positive; returns 0 if the digit doesn't exist.

  • digit_2_with_dda

    Returns the digit in the tens-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_3_with_dda

    Returns the digit in the hundreds-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_4_with_dda

    Returns the digit in the thousands-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_5_with_dda

    Returns the digit in the ten-thousands-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_6_with_dda

    Returns the digit in the hundred-thousands-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_7_with_dda

    Returns the digit in the millions-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_8_with_dda

    Returns the digit in the ten-millions-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

  • digit_9_with_dda

    Returns the digit in the hundred-millions-place of current_value as Some(digit) if it exists, or None if it doesn't. "DDA" stands for Digit Drop Accounting — it lets you tell the difference between a digit that is absent and a digit that is simply 0.

Add Methods

  • add_to_start_value

    Adds to the start_value of the ticker by the passed value; negatives subtract. The result will never overflow or wrap, staying within V::MIN to V::MAX (inclusive). If the new start_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge.

  • add_to_current_value

    Adds to the current_value of the ticker by the passed value; negatives subtract. The result will never overflow or wrap, staying within start_value to end_value (inclusive).

  • add_to_end_value

    Adds to the end_value of the ticker by the passed value; negatives subtract. The result will never overflow or wrap, staying within V::MIN to V::MAX (inclusive). If the new end_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge.

  • add_to_time_interval

    Adds to the time_interval of the ticker by the passed value; negatives subtract. time_interval can never be 0, negative, or exceed P::MAX, since that would cause .tick() to produce erratic values. This cannot be used to flip a ticker's tick direction — use .tick_up() or .tick_down() for that instead.

Percentage Methods

  • percentage_completed

    Returns the exact percentage of completion from start_value to end_value as a float. A value of 0.0 means current_value is at start_value, and 1.0 means it's at end_value. Returns None if start_value and end_value are equal.

  • percentage_remaining

    Returns the remaining percentage needed to reach end_value as a float. A value of 0.0 means current_value is at end_value, and 1.0 means it's at start_value. Returns None if start_value and end_value are equal.

Reset Methods

  • soft_reset

    Resets current_value back to start_value.

  • hard_reset

    Resets current_value back to start_value and zeroes out stored_time. Best used when you want to completely wipe whatever has been accumulated, including timing state. If you need to carry over the timing remainder from the last .tick(), do not use this.

Tick Method

  • tick

    Advances a ticker by taking in the elapsed time between two events — typically events that happen consistently and constantly, such as frame renders, though any delta time can be passed in. Adds the elapsed time to stored_time; once stored_time reaches time_interval, current_value is changed and the remainder is carried forward. Whether the change happens all at once or one unit at a time depends on is_handling_time_spikes. How current_value settles when it hits a boundary depends on the ticker's behavior — looping back to start_value, snapping to whichever boundary it hits, or freezing in place. Does nothing if the ticker is paused.

Helper Methods

  • is_mutable

    Returns true if the current behavior of the ticker is mutable, false otherwise.

  • print_information

    Prints out all the field values within a ticker.
