Tickers README

WARNING : I can be wrong, and I encourage you to do your own research before trusting my words at face-value.

Tickers is a package for Bevy projects, specifically for scenarios where you need a type to handle tracking time beyond what Bevy Timers are capable of. I created this package because Timers inside Bevy can't handle negative values without extra implementation, don't offer memory management, don't hold digit getters, and don't provide an easy way to define boundaries for the counted time. But the Tickers in this package do support the aforementioned features, and a bit more. If you feel these features would benefit you, then I encourage you to read the section "What are the Datatypes This Package Offers?" and explore the docs.rs page.

Fair warning, the docs.rs page will ultimately contain more information than this README. But that's because it's purposed to help explain things while actively coding with tickers. The README is more organized and purposed for summary, all at the cost of less explanation.

How To Use The Package?

The Recommended Way

  1. Add the package to your Rust project. 
    cargo add mirth_engine_tickers
  2. Add .add_plugin(Tickers{}) to your Bevy app.
  3. You can now make use of the datatypes that this package offers.

The "I Choose Features" Way

  1. Add the package to your project with the default features disabled, then explicitly choose which features you want.
    • ticker_reflect : Will cause all ticker types to reflect. Will add the bevy_reflect dependency and allow for the Tickers plugin to be usable.
    • ticker_systems : Will turn on all systems for every ticker type. Will add the bevy_time dependency and allow for the Tickers plugin to be usable.
    • ticker_serialize : Will make it so that ticker types can be serialized and deserialized. Will enable ticker_reflect, add the serde dependency, and allow for the Tickers plugin to be usable. 
      cargo add mirth_engine_tickers --no-default-features --features ticker_reflect, ticker_systems, ticker_serialize
  2. If you added any of the features, add .add_plugin(Tickers{}) to your Bevy app.
  3. You can now make use of the datatypes that this package offers.

The "I'll Do Everything Myself" Way

  1. Add the package to your project with the default features disabled. 
    cargo add mirth_engine_tickers --no-default-features
  2. You can now make use of the datatypes that this package offers, but you will need to create ticking systems, reflection registrations, and serialization capability yourself.

What Are The Datatypes This Package Offers?

TickersPlugin

A struct acting as a Bevy plugin used to enable ticker systems and reflections for ticker types. Use .add_plugin(Tickers{}) inside a Bevy app if you wish to make use of said systems and reflections.

Ticker

A struct used to track the time between events. Uses the types TickerValue, TickerPrecision, TickerFloatBridge, and TickerBehavior to support itself.

Holds the following fields:

  • start_value
  • current_value
  • end_value
  • time_interval
  • is_paused
  • is_ticking_up
  • is_handling_time_spikes
  • behavior

Has the following type declarations:

  • Ticker<i8, f16> : 51+ Bits
  • Ticker<i16, f16> : 83+ Bits
  • Ticker<i32, f16> : 131+ Bits
  • Ticker<i8, f32> : 91+ Bits
  • Ticker<i16, f32> : 115+ Bits
  • Ticker<i32, f32> : 163+ Bits
  • Ticker<i8, f64> : 155+ Bits
  • Ticker<i16, f64> : 179+ Bits
  • Ticker<i32, f64> : 227+ Bits

The "+" in the bit count is to recognize that the behavior field holds an enum value, and I have no idea how many bits an enum declaration represents.

Ticker Traits

TickerValue

A trait for implementing the V generic to define integer primitives a Tickercan store for its start_value, end_value, and current_value.

  • V supports i8, i16, and i32.

TickerPrecision

A trait for implementing the P generic to define float types a Ticker can use for its precision in tracking time, P impacts the time_interval and stored_time fields.

  • P supports f16, f32, and f64.
  • f16 support is coming from the crate named "half". The f16 from this crate is considered stable and the crate has over 300,000,000+ pulls; I trust it, but do your own research. From my understanding, due to how "half" has constructed their f16, an f16 value will jump up to being f32 (lossless conversion) if a device doesn't support f16.

TickerFloatBridge

A trait for implementing the F generic which grants the ability for f16, f32, and f64 to be passed in for the float fields of a Ticker constructor, no matter the precision a ticker is set to.

  • F supports f16, f32, and f64.
  • f16 support is coming from the crate named "half". The f16 from this crate is considered stable and the crate has over 300,000,000+ pulls; I trust it, but do your own research. From my understanding, due to how "half" has constructed their f16, an f16 value will jump up to being f32 (lossless conversion) if a device doesn't support f16.

Ticker Enums

TickerBehavior

An enum that establishes the different types of behavior a Ticker can use. Classifications of behavior are as follows:

  • Looper
    • Will loop when current_value hits either start_value or end_value. When a loop triggers, current_value is reset back to start_value.
  • Oneshot
    • Will assign current_value to a boundary's value if current_value were to hit start_value or end_value; start and end values are the boundaries.
    • stored_time is set to 0.0 when current_value hits end_value. This ensures the time state is completely reset once it reaches the end.

Ticker Methods

The Most Important Method

.tick(P)

Advances a ticker by taking in a passing of time between 2 events, usually for events that happen both consistently and constantly (such as when frames render); it does have the ability to take in any delta time.

If you're wondering what exactly ticking does, here is the simplified version (the specifics can be found in the doc comments):

  1. Increase stored_time by the value passed to the .tick() call (elapsed_time_between_events).
  2. If stored_time is greater than or equal to the time_interval, change current_value.
  3. Reassign stored_time to the result of stored_time %= time_interval. We do this to carry over our remainder to keep the state of time accurate to the events that are being tracked.

Ticker Constructor Methods

default()

Creates a MutLooper that has the following properties:

  • start_value is set to 0.
  • Ticks current_value from 0 to the ticker's max integer value (V::MAX).
  • end_value is set to the ticker's max integer value (V::MAX).
  • time_interval is set to 1.0.
  • Begins unpaused.
  • Will tick up.
  • Will handle time spikes.
new(V, V, V, F, bool, bool, bool, TickerBehavior)

Used for completely defining a custom ticker.

new_looper(V, V, F, bool)

Creates an unpaused Looper that ticks current_value from the supplied starting value to the passed end value.

Ticking will cause for a loop when current_value hits either boundary, resetting back to the start.

If the initial start and end values are equal, the tick direction defaults to up.

new_looper_custom(V, V, V, F, bool, bool)

Creates an unpaused Looper with fully custom starting fields.

Ticking will cause for a loop when current_value hits either boundary, resetting back to start_value.

new_oneshot(V, V, F, bool)

Creates an unpaused Oneshot that ticks current_value from the supplied starting value to the passed end value.

Ticking in a oneshot will assign current_value to whichever boundary it hits. stored_time resets to 0.0 once current_value reaches end_value.

If the initial start and end values are equal, the tick direction defaults to up.

new_oneshot_custom(V, V, V, F, bool, bool)

Creates an unpaused Oneshot with fully custom starting fields.

Ticking in a oneshot assign current_value to whichever boundary it hits; stored_time resets to 0.0 once current_value reaches end_value.

new_copy(Ticker)

Creates a copy of a passed ticker.

Useful if you want a ticker's mutability safely changed at a specific point, use this constructor to make a copy of the ticker and replace it with the copy.

new_copy_with_behavior_change(Ticker, TickerBehavior)

Creates a copy of a passed ticker with the only field change being the behavior, which is set to the passed TickerBehavior value.

Useful if you want a ticker's mutability safely changed at a specific point, use this constructor to make a copy of the ticker and replace it with the copy.

Useful if you want an immutable ticker's behavior changed, use this constructor to make a copy of the ticker and replace it with the copy.

Ticker Getter Methods

.start_value()

Returns the start_value of a ticker.

.current_value()

Returns the current_value of a ticker.

.end_value()

Returns the end_value of a ticker.

.time_interval()

Returns the time_interval of a ticker — the amount of time it takes for current_value to increase or decrease by 1.

A ticker has no built-in concept of "seconds" or any other unit; time_interval and stored_time are just two numbers compared inside .tick(), and the unit they represent depends entirely on what's passed into .tick().

.stored_time()

Returns the stored_time of a ticker — the leftover remainder from the last .tick() call, not the total elapsed time since creation or reset.

Mainly useful for debugging, logging, or custom structures that need to inspect or manually carry over a ticker's in-progress timing state.

.is_paused()

Returns true if the ticker is paused, false otherwise.

.is_unpaused()

Returns true if the ticker is unpaused, false otherwise.

.is_ticking_up()

Returns true if a ticker is set to tick its current_value up, false otherwise.

.is_ticking_down()

Returns true if a ticker is set to tick its current_value down, false otherwise.

.is_handling_time_spikes()

Returns true if the ticker can change current_value by any amount greater than or equal to 0 in a single .tick() call, catching up all built-up time at once.

Returns false if the ticker can only change current_value by 1 or 0 per .tick() call, no matter how much time has built up between calls.

.behavior()

Returns the behavior type of the ticker.

Ticker Setter Methods

.set_start_value(V)

Changes start_value to the passed value.

If the new start_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge. start_value itself cannot go outside V::MIN to V::MAX and will be clamped down if it does.

.set_current_value(V)

Changes current_value to the passed value.

current_value cannot go outside the range that start_value and end_value create and will be clamped down if it does.

.set_end_value(V)

Changes end_value to the passed value.

If the new end_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge. end_value itself cannot go outside V::MIN to V::MAX and will be clamped down if it does.

.pause()

Prevents .tick() calls on a ticker from doing their job.

.unpause()

Allows .tick() calls on a ticker to do their job.

.tick_up()

Causes the ticker's current_value to count up.

.tick_down()

Causes the ticker's current_value to count down.

.start_handling_time_spikes()

Makes it so that .tick() calls on a ticker add or subtract all built-up integer time since the last .tick() call to current_value (direction depends on is_ticking_up), with any floating remainder carried over into stored_time for the next call.

.stop_handling_time_spikes()

Makes it so that .tick() calls on a ticker only add or subtract 1 to current_value (direction depends on is_ticking_up), no matter how much time has built up between calls.

.set_behavior(TickerBehavior)

Switches the behavior of a ticker to the passed TickerBehavior type.

Does not work for tickers with immutable behavior — use the new_copy_with_behavior_change constructor to change an immutable ticker with an undesirable behavior to a ticker that has a desirable behavior. The copy can be used to substitute the original.

Ticker Equality Methods

.is_current_at_start()

Returns true if current_value and start_value are equal to one another, false otherwise.

Useful in oneshot tickers that count down to start_value to determine if the oneshot is finished.

.is_current_at_end()

Returns true if current_value and end_value are equal to one another, false otherwise.

Useful in oneshot tickers that count up to end_value to determine if the oneshot is finished.

.is_start_at_end()

Returns true if start_value and end_value are equal to one another, false otherwise.

This is mainly useful when the bounds of a mutable ticker are slowly tightening and you need to detect when they've finally met.

Ticker Difference Methods

.difference_from_start()

Returns the difference between current_value and start_value. Will only return positive numbers, including 0.

.difference_from_end()

Returns the difference between current_value and end_value. Will only return positive numbers, including 0.

.difference_from_start_to_end()

Returns the difference between start_value and end_value. Will only return positive numbers, including 0.

Ticker Digit Methods

These methods have the ability to account for whether or not a digit exists within current_value. Which is why they return an Option<i8>.

In the case that a certain digit does not exist, the value of None is returned.

.digit_1()

Returns the digit in the ones-place of current_value as Some(positive_i8).

It is impossible to return None here since the ones-place always exists.

.digit_2()

Returns the digit in the tens-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_3()

Returns the digit in the hundreds-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_4()

Returns the digit in the thousands-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_5()

Returns the digit in the ten-thousands-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_6()

Returns the digit in the hundred-thousands-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_7()

Returns the digit in the millions-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_8()

Returns the digit in the ten-millions-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

.digit_9()

Returns the digit in the hundred-millions-place of current_value as Some(positive_i8) if it exists, or None if it doesn't.

Ticker Sum Methods

.sum_to_start_value(V)

Adds to the start_value of the ticker by the passed value; negatives subtract. The result will never overflow or wrap, staying within V::MIN to V::MAX (inclusive).

If the new start_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge.

.sum_to_current_value(V)

Adds to the current_value of the ticker by the passed value; negatives subtract. The result will never overflow or wrap, staying within start_value to end_value (inclusive).

.sum_to_end_value(V)

Adds to the end_value of the ticker by the passed value; negatives subtract. The result will never overflow or wrap, staying within V::MIN to V::MAX (inclusive).

If the new end_value shifts the valid range such that current_value is left outside the boundaries, current_value is automatically clamped to the nearest valid edge.

.sum_to_time_interval(P)

Adds to the time_interval of the ticker by the passed value; negatives subtract. time_interval can never be 0, negative, or exceed P::MAX, since that would cause .tick() to produce erratic values.

This cannot be used to flip a ticker's tick direction — use .tick_up() or .tick_down() for that instead.

Ticker Percentage Methods

.percentage_completed()

Returns the exact percentage of completion from start_value to end_value as a float.

A value of 0.0 means current_value is at start_value, and 1.0 means it's at end_value. Returns None if start_value and end_value are equal.

.percentage_remaining()

Returns the remaining percentage needed to reach end_value as a float.

A value of 0.0 means current_value is at end_value, and 1.0 means it's at start_value. Returns None if start_value and end_value are equal.

Ticker Reset Methods

.soft_reset()

Resets current_value back to start_value.

.hard_reset()

Resets current_value back to start_value and zeroes out stored_time. Best used when you want to completely wipe whatever has been accumulated, including the timing state.

If you need to carry over the timing remainder from the last .tick(), do not use this.

Ticker Helper Methods

.print_information()

Prints out all the fields and their values of a ticker.
